# Phase 7 — Workspace Multi-Tenancy & UI/UX Navigation Flow

## Objectives

1. Modernize and streamline the public-to-dashboard authentication funnel (Register vs Log in).
2. Upgrade the internal data model to support a multi-tenant Workspace structure (where users collaborate on groups of brands).
3. Overhaul the Dashboard layouts to provide global context selectors (Workspace / Brand).
4. Restructure the primary Account view to reflect the most pressing operational context: today's posting schedule.

### 2. Multi-Tenant Workspace Migration (Change C)

- [x] Update `prisma/schema.prisma`:
  - Add `Workspace` model (`id`, `name`, `createdAt`, `updatedAt`).
  - Add `WorkspaceUser` link model (`userId`, `workspaceId`, `role`).
  - Modify `SocialAccount`: replace `userId` with `workspaceId`.
- [x] Create Custom Prisma Migration:
  - Run `npx prisma migrate dev --name init_workspaces --create-only`.
  - Edit the generated migration SQL to gracefully migrate existing data:
    1. For every distinct user, insert a default "Personal Workspace".
    2. Insert a link in `WorkspaceUser` for that user and new workspace.
    3. Update all `SocialAccount` rows to set `workspaceId` based on the previous `userId`.
- [ ] Apply migration (`npx prisma migrate dev`) and regenerate Prisma client.
- [x] Update backend API routes (`/api/v1/*`) and cron auth policies to validate by `workspaceId` instead of `userId`.
