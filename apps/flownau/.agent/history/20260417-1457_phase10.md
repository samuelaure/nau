# Phase 10: Pre-Flownaŭ Ingestion Bridge (Cross-Service)

## Objectives
1. **Bridge the Gap:** Flownaŭ is ready to accept captured ideas via `/api/v1/ideas/ingest`, but `9nau` does not currently forward them. Implement the webhook/event emitter in `9nau/apps/api` to push `content_idea` blocks to `flownau`.
2. **Ensure Fault-Tolerance:** The integration must survive network disconnections. `9nau` must track the sync status (`flownauSyncStatus`) of the ideas and retry if `flownau` is unreachable, so captured voice notes are never permanently lost.
3. **Robust Cross-Service Testing:** Explicitly test the integration boundary. Mock the network layer to prove that `9nau` correctly transforms an AI-triaged `content_idea` block into a payload that matches Flownaŭ's signature and uses the correct `NAU_SERVICE_KEY`.

## Pipeline Audit Summary (Context)
A comprehensive audit identified that the pipeline works successfully from `Zazŭ/Mobile -> 9naŭ Blocks -> 9naŭ AI Triage`. However, it abruptly stops there. The AI accurately tags a block as `category: 'content_idea'` with a mapped `brandId`, but the block merely sits in 9naŭ's database. There is **no mechanism** in `9nau` to forward this newly created block to Flownaŭ.
