# Phase 11: Mechanical InspoBase Synthesis & Autonomous Ideation

## Objectives
1. **Transition to Mechanical Synthesis:** Evolve Nauthenticity's InspoBase from a raw item repository into an intelligent, mechanical synthesis engine. Updates to the creative zeitgeist (synthesis) are driven strictly by the number of consumption requests from Flownaŭ, not arbitrary time schedules.
2. **Infinite Loop Prevention:** Provide Flownaŭ with a constantly evolving "Digest" of the brand's creative direction heavily weighted by recent additions and guided by long-term patterns, instead of rigid, repetitive URLs.
3. **On-Demand Ideation Trigger:** Optimize Flownaŭ's ideation cron to only activate when the pipeline has completely exhausted its pending ideas, preventing bloat.

## The Generation Mechanics
The loop is entirely mechanical:
- **Flownaŭ** requests a synthesis from Nauthenticity. Nauthenticity increments the `inspoRequestCount` for that brand.
- **Request 1 & 2:** Returns the latest cached Recent Synthesis.
- **Request 3 (and every 3rd):** Triggers a new **Recent Synthesis** using the Brand DNA, Current Global Synthesis, Previous Recent Synthesis, and any *new* posts added. If no new posts exist, it uses the previous 3 Recent Syntheses to iterate and evolve the concept.
- **Request 12 (Every 4th Recent Update):** Triggers a new **Global Synthesis** first, using Brand DNA, Previous Global Synthesis, and the last 3 Recent Syntheses. Then, it uses this fresh Global Synthesis to generate the new Recent Synthesis.
