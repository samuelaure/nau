# Phase 8: Brand-Scope UI Consolidation & Content Production Flow

## Objectives
1. **Workspace CRUD & Settings:** Introduce a Workspace Settings UI to manage the workspace and perform CRUD operations on Workspaces themselves (create new, modify current, delete), as well as perform CRUD operations on Workspace users (invite, assign roles, remove).
2. **Brand Scope Unification:** Eliminate duplicated brand dashboards. Ensure there is only *one* single-source-of-truth Dashboard for a brand.
3. **Global Page Sunset:** Completely remove global nav options and generic views for Accounts (IG Accounts), Compositions, Ideas, and Templates. Keep only: Overview, Daily Plans, Settings, and Sign Out.
4. **Daily Plan / Schedule UI Unification:** Combine the global Daily Plan UI with the brand-level Daily Schedule UI. They must use the exact same component, only differing by their data scope (All brands vs Single brand). This unified UI must retain both the day navigation and the scheduled posting slots (hours + content format).
5. **Reel Production Flow Overhaul:** Implement the definitive step-by-step Reel generation flow logic: Idea (Concept) → Template → Content + Assets → Compose → Schedule.
6. **Unified Ideation Engine:** Overhaul the Idea Generation Engine with a simplified, universal payload structural pattern.
   - **Unified Schema:** All origins use the same generation request structure, differing only by injected context: `[DNA, Strategy/Goal, Count (N), Source Concept]`.
   - **Customizable Engine Settings:** Every origin controls its own `Approval Mode` (Requires Approval vs. Auto-Approve) and `Generations Count` (How many ideas to spawn from the single trigger).
   - **Origin 1 (Captured):** Prompt fed by 9naŭ capture. Spawns `N` ideas.
   - **Origin 2 (Manual):** Prompt fed by explicit operator input. Spawns `N` ideas.
   - **Origin 3 (Automatic):** Prompt fed by retrieved InspoBase concepts. Spawns `N` ideas.
   - **Origin 4 (Replicate) [Deferred]:** Special pathway for "lightly personalized" exact replications based on original content format. Documented for future implementation.
