# Phase 12: Advanced Ideation Controls & Workspace Governance

## Objectives

1. **Granular Ideation Controls:** Bring the hidden ideation engine settings from the database to the operator's dashboard. Allow per-persona control over generation counts and approval rituals for all three origins (Captured, Manual, Automatic).
2. **Safe-Default Rituals:** Ensure no AI-generated idea is approved without human oversight by default. Reset all origin auto-approval defaults to `false`.
3. **Workspace Governance:** Implement a professional management interface for workspaces, covering workspace metadata (naming) and access control (user management/collaboration).

## Tasks

### 1. Ideation UI & CRUD Polish

- [ ] **AccountPersonas UI Extension:** Update `AccountPersonas.tsx` to include a "Generation Rituals" section.
  - Add numeric inputs for `Captured Count`, `Manual Count`, and `Automatic Count`.
  - Add explicit checkboxes for `Captured Auto-Approve`, `Manual Auto-Approve`, and `Automatic Auto-Approve`.
- [ ] **Manual Override:** Update the "Manual Concept Idea" modal in `AccountIdeas.tsx`. Add a numeric input for "Number of ideas to spawn" which overrides the persona's `manualCount` for that specific session.
- [ ] **Data Sync:** Update the `PUT /api/personas/[id]` and `POST /api/personas` routes to strictly map these 6 new fields.
- [ ] **Idea Text Editing:**
  - Update `PUT /api/ideas/[id]` on the backend to allow updating `ideaText`.
  - Add an "Edit" action to idea cards in `AccountIdeas.tsx`.
  - Implement an "Edit Idea" modal to allow the operator to refine AI-generated or captured text before approval.
- [ ] **Bulk Actions:** Add "Approve All Pending" and "Clear All Used" buttons to the Content Backlog to speed up workflow.

### 2. Default Hardening 🛡️

- [ ] **Schema Refactor:** Update `prisma/schema.prisma` to set `capturedAutoApprove` and `manualAutoApprove` defaults to `false`.
- [ ] **Logic Audit:** Update the idea ingestion and generation services to ensure they strictly respect the Persona settings, falling back to `false` if settings are missing.

### 3. Workspace Management UI

- [ ] **Workspace Settings Route:** Create `src/app/dashboard/workspace/[workspaceId]/settings/page.tsx`.
- [ ] **Workspace Metadata Form:** Build a form to update the Workspace name and delete the workspace.
- [ ] **Member Management:** Implement a "Collaborators" section:
  - List all users tied to the workspace via `WorkspaceUser`.
  - Add/Invite secondary users to the workspace.
  - Remove users from the workspace.
- [ ] **Dashboard Integration:** Add a "Workspace Settings" link to the dashboard sidebar or the workspace overview header.

## Verification Criteria

- [ ] A new Persona created for a brand defaults to `Manual Auto-Approve: false`.
- [ ] Changing the `Manual Count` to `10` in a Persona results in the Brainstorm modal defaulting to `10`, but allows the user to change it to `2` for a specific session.
- [ ] A non-owner user can be added to a workspace and only see the brands within that specific workspace.
- [ ] Changing a Workspace name is reflected immediately across the breadcrumbs and sidebar.
